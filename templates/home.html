Pages

Register

If the locale provided is already present in the in the UsersLocales table, the user should not be permitted to register with that localeName.

The registration page adds a new user to the Users table -- firstName, lastName, and an autoincremented userID. If the locale provided is not yet in the Locales table, it is added -- localeName and an autoincremented localeID. Finally, an entry is added to the UsersLocales table linking the user and locale.

Login

The login page uses a select statement to find the user's userID (based on the first and last name they logged in with), which is used to find the user's name, locale, and upcoming and available walks and activities.

Quarpedia

This is the user's main page, where they can see their current activity.

To change any of this content, the user needs to navigate to other links:
  • locales to add locales
  • walks to change the user's current locale
  • activities to add activities to do

Locales

This is simply the locales table. New locales can be added here.

Walks

Walks inserts to UsersLocales by providing the user with a dropdown of destinations they can reserve. These (locales) are filtered to only locales no one else is currently at (locales not in UsersLocales). This also results in a delete of the current UserLocale relationship.

Activities

Activities inserts to UsersLocales by providing the user with a dropdown of destinations they can reserve. These (locales) are filtered to only locales no one else is currently at (locales not in UsersLocales). This also results in a delete of the current UserLocale relationship.
