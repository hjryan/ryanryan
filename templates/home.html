Pages


Register

The registration page adds a new locale to the Locales table (localeName and an autoincremented localeID) if the locale does not yet exist. If the locale already exists in the Users table, the user will not be permitted to register there. Then the new user is added to the Users table -- firstName, lastName, an autoincremented userID, and the localeID that was just added.


Login

The login page uses a select statement to find the user's userID (based on the first and last name they logged in with), which is used to find the user's name, locale, and upcoming and available walks and activities.


Quarpedia

This is the user's main page, where they can see their current activity.

To change any of this content, the user needs to navigate to other links:
  • locales to add locales
  • walks to change the user's current locale
  • activities to add activities to do


Locales

This is simply the locales table. New locales can be added, and existing locales updated here.


Walks

Walks updates the user's localeID in the Users table by providing the user with a dropdown of destinations they can reserve -- these (locales) are the locales no one else is currently at (locales with localeIDs not in Users). The user names this walk, and it is added to the Walks table.


Activities

The activities page allows users to add an activity to the Activities table, the ActivitiesLocales table, and the ActivitiesUsers table (ActivitiesUsers will only allow the user to book Activities in their locale, as determined by ActivitiesLocales).
